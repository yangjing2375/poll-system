## 一、项目概述

本次实训项目为基于 **PHP + MySQL** 开发的在线投票系统，完整实现用户注册登录、自定义创建投票（支持文字、图片两种投票类型）、用户在线投票、投票数据可视化结果查看、管理员后台全局管理等核心功能。目前项目已完成线上部署，可正常访问运行，系统访问地址：**http://poll-system.freepage.cc**。

系统创新性搭建了完整的专题投票体系，细分十大热门投票专题，具体包含：

| 专题类别 | 涵盖场景 |
|----------|----------|
| 家庭&情感 | 家庭关系、情感话题 |
| 美食专题 | 美食推荐、饮食偏好 |
| 校园专题 | 校园活动、学习生活 |
| 职场专题 | 工作相关、职业发展 |
| 影视专题 | 电影电视、娱乐作品 |
| 运动&出行 | 体育活动、旅行出行 |
| 娱乐专题 | 娱乐八卦、休闲活动 |
| 旅游专题 | 旅游目的地、旅行体验 |
| 汽车专题 | 汽车品牌、车型选择 |
| 游戏专题 | 游戏推荐、电竞话题 |

平台支持用户自由创建各类型专题投票、参与公开投票、查看可视化投票结果，同时可查看个人全部投票记录，适配校园活动调研、趣味互动投票、大众偏好统计等多元化使用需求，整体架构完整、功能完善、运行稳定。

## 二、项目开发难点及解决过程

### 困难一：数据库表结构设计与外键约束问题

**问题描述**：在数据库搭建阶段，设计多表关联结构时，Polls投票表需要同时关联普通用户表（users）和管理员表（admins），用于记录投票创建者信息。但数据库外键约束仅支持单表关联，无法同时引用两张数据表，直接创建外键约束会报错，导致表结构搭建失败。

**解决过程**：

1. 调整表结构设计，移除polls表中对users表的外键约束，允许管理员和普通用户都能创建投票；
2. 引入数据库事务机制，对投票创建、删除、数据更新等核心操作进行事务包裹，保障多表数据一致性，避免数据错乱、缺失；
3. 编写标准化建表脚本 `create_poll_tables.sql`，同时迭代更新多个数据库迁移脚本，补充新增字段、优化字段属性，完成数据库结构迭代优化。

---

### 困难二：PHP本地开发环境配置与服务器搭建故障

**问题描述**：使用phpstudy_pro搭建本地开发服务器时，默认80端口被本地其他程序占用，导致网站无法正常启动访问，数据库连接也存在配置不匹配问题。

**解决过程**：

1. 修改服务器运行端口，将默认80端口替换为8080，规避端口冲突问题；
2. 同步修改项目`.env`配置文件，更新项目访问地址为 `APP_URL=http://localhost:8080`；
3. 核对MySQL数据库账号、密码、端口配置，修正配置参数错误，完成数据库连接调试；
4. 重启本地服务，多次测试网站访问、数据读写功能，确保开发环境稳定可用。

### 困难三：图片投票上传、压缩与存储逻辑实现复杂

**问题描述**：系统需支持图片类型投票选项，原生图片上传存在体积过大、加载缓慢、存储占用高的问题，且前后端图片传输、解析、存储逻辑衔接难度较高。

**解决过程**：

1. 前端基于FileReader API读取本地图片文件，结合Canvas画布实现图片压缩处理，限定图片最大尺寸为**300×300**、压缩质量**70%**，统一转换为JPEG格式，减小图片体积；
2. 将压缩后的图片转换为Base64编码格式，以JSON数据形式传输至后端接口；
3. 后端接收Base64数据，校验合规性后存入数据库`option_image`字段；
4. 测试不同尺寸、格式的图片上传场景，优化压缩逻辑，保证图片清晰度的同时，提升上传和加载效率。

**核心代码示例**（前端图片压缩）：

```javascript
function compressImage(file) {
    return new Promise((resolve) => {
        const reader = new FileReader();
        reader.onload = (e) => {
            const img = new Image();
            img.onload = () => {
                const canvas = document.createElement('canvas');
                const ctx = canvas.getContext('2d');
                const maxSize = 300;
                let width = img.width;
                let height = img.height;
                
                if (width > height) {
                    if (width > maxSize) {
                        height *= maxSize / width;
                        width = maxSize;
                    }
                } else {
                    if (height > maxSize) {
                        width *= maxSize / height;
                        height = maxSize;
                    }
                }
                
                canvas.width = width;
                canvas.height = height;
                ctx.drawImage(img, 0, 0, width, height);
                resolve(canvas.toDataURL('image/jpeg', 0.7));
            };
            img.src = e.target.result;
        };
        reader.readAsDataURL(file);
    });
}
```

### 困难四：投票数据统计图表可视化渲染异常

**问题描述**：管理员后台需实现数据可视化功能，包含投票趋势、热度排行、专题分布等图表展示，开发中出现图表初始化失败、数据渲染延迟、明暗模式适配异常等问题。

**解决过程**：

1. 引入**Chart.js**可视化插件，分别开发折线图（7日投票趋势）、柱状图（投票热度排行）、饼图（专题投票分布）三类统计图表；
2. 规范后端API接口返回格式，统一封装系统概览、趋势数据、热门投票、活跃用户等统计数据；
3. 优化前端渲染逻辑，通过异步请求回调函数，确保DOM节点加载完成后再初始化图表，解决渲染时机错误问题；
4. 适配系统日夜切换模式，优化图表配色、透明度样式，保证不同主题下图表正常显示。

---

### 困难五：用户会话管理与权限控制失效

**问题描述**：使用PHP Session管理用户登录状态时，存在跨页面会话丢失、登录状态失效问题，且普通用户与管理员权限区分不清晰，部分接口存在权限越权风险。

**解决过程**：

1. 基于PHP Session搭建登录认证体系，区分普通用户、管理员双重权限体系，前端通过localStorage缓存用户信息，实现页面状态共享；
2. 排查会话丢失原因，在前端fetch请求中添加 `credentials: 'include'` 参数，保证跨页面请求携带Session Cookie；
3. 对所有核心API接口增加权限校验逻辑，未登录用户、低权限用户禁止访问管理员功能页面及接口，完善系统权限壁垒。

---

### 困难六：前后端接口联调数据交互异常

**问题描述**：前后端联调阶段，因页面目录层级不同，API请求路径出错，同时存在跨域问题、数据格式不统一，导致接口请求失败、数据无法正常交互。

**解决过程**：

1. 梳理项目目录结构，统一前端API请求相对路径，适配views页面层级路由规则；
2. 在后端`db.php`公共配置文件中添加CORS跨域响应头，彻底解决跨域请求拦截问题；
3. 通过浏览器开发者工具监控接口请求、响应数据，排查参数格式、编码问题，统一前后端数据交互规范，重点适配图片Base64编解码逻辑。

---

### 困难七：前端CSS样式兼容性与响应式适配问题

**问题描述**：前端页面在不同浏览器、不同设备屏幕下样式错乱，日夜主题切换时部分元素样式失效，卡片布局、投票选项排版适配异常。

**解决过程**：

1. 采用**Flexbox+Grid**弹性布局重构页面结构，实现全页面响应式适配，适配电脑、平板等不同屏幕尺寸；
2. 规范化日夜双主题样式代码，为所有页面元素配置两套独立样式，保证主题切换无缝生效；
3. 兼容Chrome、Edge、Firefox等主流浏览器，逐一调试样式bug，统一页面展示效果。

---

### 困难八：管理员账号登录认证失败

**问题描述**：项目初期普通用户登录正常，但管理员账号始终提示"用户名或密码错误"，无法登录后台。

**解决过程**：

1. 排查数据库`admins`管理员表数据，发现密码字段存储的是加密哈希值，并非明文密码；
2. 核对数据库初始化脚本，确认管理员哈希密码对应的原始明文为`password`，而非预设的`admin123`；
3. 使用正确明文密码完成登录，同时记录问题根源，规范后续账号密码配置流程，避免参数混淆。

---

### 困难九：线上部署数据库字段缺失导致功能异常

**问题描述**：项目本地运行正常，部署至线上服务器后，创建投票功能提示创建失败，无法正常提交。

**解决过程**：

1. 后端添加日志记录功能，捕获SQL执行报错信息，定位问题为线上数据库字段缺失；
2. 对比本地与线上数据库表结构，确认线上`polls`表缺失`topic`、`is_hot`、`is_anonymous`、`option_type`等字段，`poll_options`表缺失`option_image`字段；
3. 在线上phpMyAdmin中执行ALTER TABLE语句，批量补充缺失字段；
4. 优化图片存储字段，将原`VARCHAR(500)`类型修改为`MEDIUMTEXT`，适配Base64超长图片数据存储；
5. 移除外键约束限制，适配管理员创建投票业务场景，测试所有功能恢复正常。

---

### 困难十：图片投票选项前端无法正常展示

**问题描述**：后台可正常存储图片投票数据，但前端专题投票页面无法加载、展示图片选项，仅文字投票正常显示。

**解决过程**：

1. 通过浏览器开发者工具查看 `get_poll_results.php` 的响应，确认`option_type`为 `"image"`，`option_image`有完整的Base64数据；
2. 对比 `index.html` 和 `topic_polls.html` 的投票模态框代码，发现专题页面缺少图片显示逻辑；
3. 在 `topic_polls.html` 的 `openVoteModal` 函数中添加了图片选项的处理逻辑：

```javascript
const optionType = data.data.poll.option_type || 'text';
if (optionType === 'image' && option.option_image) {
    optionContent = `<img src="${option.option_image}" class="vote-option-image" alt="${option.option_text}">`;
}
```

4. 添加了相应的CSS样式确保图片正确显示；
5. 重新部署文件，多次测试图片投票创建、展示、投票全流程，功能运行正常。
